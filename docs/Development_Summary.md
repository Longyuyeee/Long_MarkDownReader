## 1. 当前项目状态

项目已跨越 v0.4.8 里程碑，核心目标是提升大规模文档库下的导航效率与多场景切换的响应速度。本次更新彻底攻克了侧边栏状态同步滞后、跨平台路径兼容折叠及图片渲染延迟等交互痛点。

### 已实现核心功能 (v0.4.8 增量)：

- **托盘交互体系 2.0 (Tray Reborn)**：
  - **行为解耦**：彻底分离托盘左/右键逻辑。左键单击实现“瞬时唤醒+窗口夺焦”，右键仅弹出功能菜单，解决了 Windows 任务栏交互混乱的问题。
  - **原子唤醒链**：在 Rust 端通过 `unminimize` -> `show` -> `set_focus` 的组合调用，确保窗口在任何挂起状态下都能强行置顶。
- **侧边栏深度同步系统 (Intelligent Navigation Sync)**：
  - **路径自适应解析**：引入 Separator 自动识别逻辑，完美兼容 Windows (`\`) 和 Unix (`/`) 路径，解决了在特定 OS 下文件夹无法折叠的顽疾。
  - **API 驱动滚动**：利用 `treeInst.scrollTo` 替代不可靠的 DOM 操作，实现标签页切换时左侧树状列表的毫秒级平滑定位。
- **极致性能优化 (Zero-Lag Engine)**：
  - **内存快照缓存**：在 Pinia 层引入文档内容 LRU 缓存，已打开标签页的二次切换达到“零 I/O”级别的瞬时呈现。
  - **正则路径预修复**：弃用渲染后扫描 DOM 的旧模式，在将源码交给 Vditor 前通过正则瞬间完成协议转换，消除了图片加载时的闪烁感。
- **现代化视觉增强 (Premium Visuals)**：
  - **悬浮统计预览**：鼠标悬停文档时弹出基于 `backdrop-filter` 的高强度毛玻璃统计卡片，实时显示创建/修改时间及物理大小。
  - **智能视口防御**：悬浮窗内置边界检测算法，根据光标位置自动翻转布局，确保内容 100% 不会被屏幕切断。
- **主题联动闭环**：全局颜色主题切换时，编辑器背景色、代码高亮风格、标签页样式及设置预览区实现全自动同步。

| :------------------------------ | :---------------------------------------------------------------------------- | :---------------------------------------------------------------------------------- |
| **文件夹点击无法折叠**    | 之前的 Watcher 逻辑在设置 `expandedKeys` 时强行使用 `/` 替换了原始的 `\` 路径键。 | **自适应分隔符**：动态提取路径特征，确保展开列表中的 Key 与树节点 Key 绝对对齐。 |
| **打字过程中偶发顿挫**    | `input` 回调中包含了全量扫描 DOM 图片并调用 IPC 的逻辑，导致主线程周期性阻塞。 | **策略转移**：移除 `input` 扫描，改为加载时处理 + 修改后增量处理。              |

  - **精准按需同步**：导入 MD 到库时，软件会自动扫描并仅拷贝文档引用的图片，杜绝资源文件夹冗余。

  - **绿色卸载清理**：删除文档时同步卸载库内关联图片，实现物理空间的自动化清理。
