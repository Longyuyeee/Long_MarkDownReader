# 胧编辑 · md助手 开发阶段总结 (2026-03-04 - 大纲与同步重构版)

## 1. 当前项目状态
项目已成功实现高度稳定的右侧大纲同步系统与可靠的影子副本存档逻辑。通过“推倒重构”方案，彻底解决了 Vditor 外部大纲渲染空白及多文档并发保存冲突的疑难杂症。

### 已实现核心功能：
- **实时大纲系统 (Manual Extraction)**：
    - **原理**：放弃 Vditor 官方 `outlineRender`，改用原生 DOM 扫描 + `MutationObserver` 监听。
    - **亮点**：自动为缺失 ID 的标题注入锚点，支持 H1-H6 阶梯缩进，毫秒级响应输入变化。
- **影子副本 (Shadow Copy) 3.0**：
    - **路径校验锁**：引入 `lastLoadedPath` 原子锁，确保自动保存的内容与当前激活的文件 100% 匹配。
    - **内容归一化**：比对时自动忽略 Windows/Unix 换行符差异，杜绝冗余气泡。
- **知识库智能关联**：修复了启动时因配置加载延迟导致的“未关联”显示 Bug。

## 4. 主题同步与系统深度集成 (2026-03-04)
修复了多主题切换中的背景同步失效问题，并完成了快速捕获小窗与系统右键菜单的集成。

### 已实现核心功能：
- **配置持久化体系 (AppConfig)**：
    - **后端保障**：Rust 端引入 `Default` 特性与 `rename_all = "camelCase"`，确保配置文件即使在字段缺失时也能平滑升级，并与前端键名完美对齐。
    - **前端防御**：Store 端重构 `updateConfig`，采用非覆盖式合并逻辑，防止设置页面因字段不全而导致配置“黑洞”。
    - **实时捕获**：编辑器端引入点击嗅探机制，解决 Vditor 工具栏按钮操作不触发 `input` 事件导致的模式丢失问题。

### 3. 待解决的挑战 (Critical Issues)
- **【核心遗留】Vditor 状态恢复失效**：
    - **现象**：虽然“修改背景颜色”能完美保存并恢复，但“代码高亮风格”和“分栏/预览模式”在软件重启后依然会回退到默认状态。
    - **已尝试路径**：
        1. **后端对齐**：Rust 端已实现 `camelCase` 键名对齐与 `Default` 容错。
        2. **前端监听**：已增加编辑器容器 `click` 嗅探，试图在非输入状态下捕获工具栏操作。
        3. **初始化强制加载**：在 `onMounted` 阶段 `await` 配置加载，并在 `initVditor` 时传入 Store 值。
    - **疑点分析**：Vditor 内部的状态管理（尤其是预览模式 `both`）与其初始化 `mode` 并非同一概念，且 Vditor 可能在 `after` 回调后又执行了某种内部重置逻辑，覆盖了 Store 注入的值。需后续深入 Vditor 内部 `options` 树或考虑在渲染完成后进行“二次强制设置”。
- **大纲点击定位偏差**：在极长文档中，由于顶部固定栏遮挡，定位需向上偏移约 40px。
