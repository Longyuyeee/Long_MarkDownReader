# 胧编辑 · md助手 开发阶段总结 (2026-03-04 - 大纲与同步重构版)

## 1. 当前项目状态
项目已成功实现高度稳定的右侧大纲同步系统与可靠的影子副本存档逻辑。通过“推倒重构”方案，彻底解决了 Vditor 外部大纲渲染空白及多文档并发保存冲突的疑难杂症。

### 已实现核心功能：
- **实时大纲系统 (Manual Extraction)**：
    - **原理**：放弃 Vditor 官方 `outlineRender`，改用原生 DOM 扫描 + `MutationObserver` 监听。
    - **亮点**：自动为缺失 ID 的标题注入锚点，支持 H1-H6 阶梯缩进，毫秒级响应输入变化。
- **影子副本 (Shadow Copy) 3.0**：
    - **路径校验锁**：引入 `lastLoadedPath` 原子锁，确保自动保存的内容与当前激活的文件 100% 匹配。
    - **内容归一化**：比对时自动忽略 Windows/Unix 换行符差异，杜绝冗余气泡。
- **知识库智能关联**：修复了启动时因配置加载延迟导致的“未关联”显示 Bug。

## 4. 主题同步与系统深度集成 (2026-03-04)
修复了多主题切换中的背景同步失效问题，并完成了快速捕获小窗与系统右键菜单的集成。

### 已实现核心功能：
- **主题系统深度同步**：
    - **属性上移**：通过 Vue `watch` 强制将 `data-theme` 同步至 `document.body`，彻底解决背景颜色变量无法渗透至父级元素的问题。
- **快速捕获 (Quick Capture)**：
    - **独立窗口**：通过 Tauri 多窗口机制实现悬浮小窗，支持毛玻璃效果与独立关闭。
    - **Inbox 存入逻辑**：内容自动保存至当前库根目录下的新文档，支持保存后自动关闭。
- **系统右键菜单集成**：
    - **注册表脚本 (PS1)**：通过 Rust 后端调用 PowerShell 脚本注入注册表，实现 `.md` 文件右键关联“使用胧编辑打开”。
- **长文本排版优化**：
    - **文件树与标签页截断**：应用 `ellipsis` 智能截断，保持单行布局美观。
- **标签导航自动化**：
    - **自动居中滚动**：实现 `activeTab` 自动对齐功能，切换标签时自动滚动至可视区域中心。

### 3. 待解决的挑战
- **大纲点击定位偏差**：在极长文档中，由于顶部固定栏遮挡，定位需向上偏移约 40px。
- **搜索结果高亮**：全文搜索功能尚未与编辑器内容联动高亮。
