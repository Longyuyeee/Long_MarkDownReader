# 胧编辑 · md助手 开发阶段总结 (2026-03-04 - 大纲与同步重构版)

## 1. 当前项目状态
项目已成功实现高度稳定的右侧大纲同步系统与可靠的影子副本存档逻辑。通过“推倒重构”方案，彻底解决了 Vditor 外部大纲渲染空白及多文档并发保存冲突的疑难杂症。

### 已实现核心功能：
- **实时大纲系统 (Manual Extraction)**：
    - **原理**：放弃 Vditor 官方 `outlineRender`，改用原生 DOM 扫描 + `MutationObserver` 监听。
    - **亮点**：自动为缺失 ID 的标题注入锚点，支持 H1-H6 阶梯缩进，毫秒级响应输入变化。
- **影子副本 (Shadow Copy) 3.0**：
    - **路径校验锁**：引入 `lastLoadedPath` 原子锁，确保自动保存的内容与当前激活的文件 100% 匹配。
    - **内容归一化**：比对时自动忽略 Windows/Unix 换行符差异，杜绝冗余气泡。
- **知识库智能关联**：修复了启动时因配置加载延迟导致的“未关联”显示 Bug。

## 3. 多主题与视觉增强版 (2026-03-04)
成功构建了全新的全局主题变量系统，并解决了背景色同步与深色模式对比度的核心痛点。

### 已实现核心功能：
- **六大主题色系 (Multi-Theme System)**：
    - **色系选择**：纯白、护眼绿、清爽蓝、浪漫粉、深色、跟随系统。
    - **底层逻辑**：通过 Vue `watch` 实时将主题状态同步至 `document.body` 的 `data-theme` 属性，利用全局 CSS 变量实现毫秒级无缝换肤。
- **设置界面交互升级**：
    - **模块穿透**：彻底消除硬编码背景色，所有卡片、输入框、单选按钮均基于 `--theme-card` 变量，确保与主背景完美融合。
    - **物理动效**：引入分层进场动画（Grid-item Slide-up）与物理按压反馈 (`scale(0.96)`)。
- **视觉修复**：
    - 修复了深色模式下文字对比度低的问题（提升至 `#f5f5f7`）。
    - 解决了全局背景色无法响应 CSS 变量更新的属性作用域 Bug。

### 2. 踩坑点总结 (The Hard Way)
| 坑点场景 | 根本原因 | 最终处理方式 |
| :--- | :--- | :--- |
| **背景色不改变** | `data-theme` 定义在子容器上，`body` 无法向上读取 CSS 变量。 | **属性上移**：通过 Watch 强制将属性挂载到 `document.body`。 |
| **运行时 ReferenceError** | 在 `App.vue` 中使用了 `watch` 但未引入。 | **规范引入**：补全 Vue 核心 API 的解构引入。 |
| **Naive UI 残留白块** | 组件库内置背景色优先级高于 CSS 变量。 | **主题覆盖**：在 `themeOverrides` 中将 `cardColor` 等指向变量。 |
