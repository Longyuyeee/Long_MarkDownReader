## 2. 踩过的坑点与解决方案 (Pitfalls)

### 2.1 侧边栏拖动性能癌症
- **现象**：鼠标拖动分隔条时，侧边栏移动有 400ms 的明显滞后，极度不跟手。
- **原因**：为了美观设置了 `transition: width 0.4s`。拖拽时每一像素的宽度改变都会触发一次 0.4s 的补间动画，导致 CPU 负载飙升且视觉延迟。
- **对策**：在 `is-dragging` 状态下，使用 `:deep(.is-dragging, .is-dragging *) { transition: none !important; }` 彻底切断所有动画计算。

### 2.2 Vditor 大纲“回退”机制
- **现象**：即使指定了外部容器，大纲依然反复在编辑器中间区域出现。
- **原因**：Vditor 初始化时若目标容器不可见或宽度为 0，会默认触发 Fallback 机制回退到内置侧边栏。
- **对策**：
    1.  物理禁用：`outline: { enable: false }`。
    2.  静态渲染：改用 `Vditor.outlineRender(content, target)` 主动驱动。
    3.  CSS 抹除：使用样式强制隐藏编辑器内部的 `.vditor-outline`。

### 2.3 Vue 模板函数丢失 Bug
- **现象**：全量写入代码后，点击侧边栏报错 `startResizing is not a function`。
- **原因**：在大段代码写入时，若脚本部分未正确闭合或定义顺序错误，Vue 编译器可能无法将函数正确挂载到实例作用域。
- **对策**：确保 `script setup` 中所有与模板交互的函数均在脚本顶部定义，并保持逻辑块的原子性。

## 3. 当前遗留问题 (Technical Debt)

### 3.1 右侧大纲渲染稳定性
- **现状**：虽然已切换到静态渲染模式，但由于 Vditor 异步加载机制，`Vditor.outlineRender` 往往在内容未完全就绪时调用，导致渲染结果为空。
- **计划**：下一步将引入 MutationObserver 监听编辑器 DOM 变化，实现真·响应式大纲同步。

### 3.2 跨平台路径斜杠一致性
- **现状**：目前大量使用 `lastIndexOf('\\')`，这在 Linux/macOS 下会失效。
- **计划**：统一引入 `@tauri-apps/api/path` 或在 Rust 后端进行标准规范化处理。
