# Tauri 单实例模式下的窗口唤醒与夺焦优化

> **技术关键词**：Tauri 2.0, 单实例 (Single Instance), 窗口管理, 自动唤醒, 用户交互

在单实例模式下，处理从外部系统（如资源管理器）传入的文件打开请求时，如何正确“跳”到用户面前是一个关键细节。

### 1. 现象描述
当“胧编辑”已在后台或最小化运行时，通过右键“打开方式”尝试打开新文件，程序虽然正确处理了文件逻辑，但主窗口依然保持最小化，用户必须手动点击托盘才能看到界面。

### 2. 原因分析
Tauri 的单实例插件在转发命令行参数时，只负责触发事件监听。默认情况下，Windows 为了防止应用“抢夺焦点”，不会在接收到事件后自动将窗口置顶或还原。

### 3. 解决方案
在 `open-file` 事件的回调函数中显式加入窗口操作指令：
1. **状态还原**：调用 `win.unminimize()` 确保窗口从任务栏恢复。
2. **强制显示**：调用 `win.show()` 处理可能的隐藏状态（如点击了关闭到托盘）。
3. **夺取焦点**：调用 `win.set_focus()` 或前端对应的聚焦方法，确保窗口位于最上层。

### 4. 总结
良好的系统级集成不仅是逻辑正确，更是交互反馈的闭环。在多任务切换场景下，显式的窗口唤醒逻辑是保障软件“响应感”的最后一步。
