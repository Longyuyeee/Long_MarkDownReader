# Vue 3 深度监听下的配置持久化死循环防护

> **技术关键词**：Vue 3, watch, deep, 响应式系统, 配置持久化, 逻辑死循环

在实现配置界面的实时自动保存时，极易陷入数据流的死循环陷阱。

### 1. 现象描述
在设置页面中，一旦进入界面，程序会陷入疯狂的保存日志刷新中，CPU 占用率飙升，UI 响应变得极其迟钝。

### 2. 原因分析
该问题源于“数据加载”与“变更监听”共用了同一个响应式对象。
- 当组件挂载时，执行 `loadConfig` 异步获取后端数据。
- 数据的填入触发了 `watch(config, ..., { deep: true })`。
- `watch` 触发 `updateConfig` 调用后端接口进行保存。
- `updateConfig` 的某些回写逻辑或状态更新可能再次触发 `watch`，从而形成了闭环自激。

### 3. 解决方案
引入 **初始化锁 (Initialization Lock)** 机制：
1. **状态定义**：定义 `isInitializing = ref(true)`。
2. **逻辑切断**：在 `watch` 回调的首行判断该锁，若为 `true` 则直接中断保存逻辑。
3. **时机释放**：在 `onMounted` 的数据填充完成后，利用 `nextTick` 在下一个 DOM 更新周期释放该锁。

### 4. 经验总结
在处理“加载即触发”的联动逻辑时，显式的状态锁比复杂的防抖（Debounce）更具确定性。它能确保系统的响应式行为完全由“用户意图”而非“初始化行为”驱动。
