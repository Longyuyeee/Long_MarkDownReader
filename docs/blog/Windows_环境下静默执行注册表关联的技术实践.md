# Windows 环境下静默执行注册表关联的技术实践

> **技术关键词**：Windows 注册表, PowerShell, HKCU, CREATE_NO_WINDOW, 文件关联

实现 Windows 平台的文件关联（默认编辑器）需要平衡“用户权限”与“交互优雅度”。

### 1. 现象描述
旧版关联功能存在两个问题：一是写入时因权限不足被系统拦截；二是每次执行都会瞬间弹出一个控制台黑框，产生强烈的视觉干扰。

### 2. 原因分析
- **权限问题**：`HKEY_CLASSES_ROOT` 需要管理员权限，且在 Windows 11 下其优先级低于用户的 `UserChoice` 偏好。
- **视觉干扰**：Rust 的 `Command::new("powershell")` 在 Windows 下默认会创建新的控制台窗口。

### 3. 解决方案
1. **路径优化**：改写 `HKEY_CURRENT_USER\Software\Classes`。此路径具有更高的用户优先级，且普通用户权限即可写入。
2. **静默执行**：引入 Windows 特有的进程创建标志 `0x08000000` (CREATE_NO_WINDOW)，确保脚本在后台无感运行。
3. **默认值写入规范**：在 PowerShell 中，注册表默认键值的写入不能使用空字符串 Name，必须使用特定的 `Set-Item` 语法。

### 4. 经验总结
桌面软件的系统级交互应追求“无感化”。通过合理利用用户级注册表权限和进程标志位，可以在不骚扰用户的前提下完成复杂的系统集成。
