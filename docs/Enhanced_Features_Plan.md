# 胧编辑·MD助手 增强功能规划 (2026-03-05)

## 📋 文档概述

本文档基于项目现有功能，提出一系列增强功能建议，旨在将胧编辑从优秀的Markdown编辑器升级为**生产级知识管理平台**。所有建议均考虑技术可行性和用户价值。

---

## 🎯 核心升级方向

### 1. **智能化** - 集成AI能力，提升写作效率
### 2. **知识化** - 强化内容关联，构建知识网络
### 3. **专业化** - 增强技术集成，服务专业用户
### 4. **自动化** - 优化工作流程，减少重复劳动

---

## 🔥 第一梯队：立即可以实施（高性价比）

### 1.1 文件编码检测与转换 ⭐⭐⭐⭐⭐
**用户需求**：处理不同编码的历史文档，避免乱码问题
**功能描述**：
- 自动检测文件编码（UTF-8、GBK、GB2312、BIG5、Shift-JIS等）
- 状态栏显示当前编码
- 右键菜单支持编码转换
- 批量转换文件夹内所有文件

**技术实现**：
```rust
// Rust后端接口
#[tauri::command]
async fn detect_encoding(path: String) -> Result<String, String>
#[tauri::command]
async fn convert_encoding(path: String, target: String) -> Result<(), String>
```

**优先级**：P0（用户提出，解决实际问题）

### 1.2 AI智能写作助手 ⭐⭐⭐⭐⭐
**用户需求**：提升写作效率，获得智能辅助
**功能描述**：
- 集成Claude/OpenAI/Gemini API
- 选中文本右键AI操作菜单
  - ✍️ 重写/润色
  - 📝 扩展/续写
  - 🌐 翻译（支持中英日等）
  - 🔍 总结/提取要点
  - 💡 生成标题/大纲
- 独立AI对话侧边栏
- API密钥安全存储

**技术实现**：
- 设置面板添加AI配置
- 编辑器工具栏AI按钮
- 流式响应显示

**优先级**：P0（核心差异化功能）

### 1.3 模板系统 ⭐⭐⭐⭐⭐
**用户需求**：快速创建标准化文档
**功能描述**：
- 内置模板库（日记、会议记录、周报、技术文档等）
- 用户自定义模板（保存到`~/.misty/templates/`）
- 新建文件时选择模板
- 模板变量支持（{{date}}、{{title}}等）

**技术实现**：
- 模板文件使用`.md.template`扩展名
- 变量替换引擎
- 模板管理界面

**优先级**：P1

### 1.4 字数统计与阅读时间 ⭐⭐⭐⭐
**用户需求**：了解文档规模和阅读耗时
**功能描述**：
- 实时显示中文字数、英文字数、总字数
- 估算阅读时间（中文300字/分钟，英文200词/分钟）
- 编辑器底部状态栏集成
- 文档统计面板（段落数、标题数等）

**技术实现**：
- 防抖字数统计
- 国际化阅读速度配置

**优先级**：P1（快速见效）

### 1.5 最近访问文件 ⭐⭐⭐⭐
**用户需求**：快速回到最近工作文档
**功能描述**：
- 侧边栏"最近"分组显示最近20个文件
- 命令面板(Ctrl+P)输入`@recent`显示最近文件
- 支持清除访问记录
- 显示最后访问时间

**技术实现**：
- 存储到本地索引
- 基于文件路径去重

**优先级**：P1

## 🚀 第二梯队：知识管理增强

### 2.1 双向链接与知识图谱 ⭐⭐⭐⭐⭐
**用户需求**：建立文档间关联，形成知识网络
**功能描述**：
- `[[文档名]]`语法创建双向链接
- 右侧面板显示"链接关系图"
  - 当前文档的**出链**（引用的文档）
  - 当前文档的**反链**（引用此文档的文档）
- 可视化图谱展示（使用d3.js或类似库）
- 悬停预览链接文档内容

**技术实现**：
- 建立文档索引关系
- 定期扫描更新链接
- 可视化组件

**价值**：将编辑器升级为知识管理工具

### 2.2 文档属性（Front Matter）系统 ⭐⭐⭐⭐
**用户需求**：为文档添加结构化元数据
**功能描述**：
- YAML Front Matter支持
```yaml
---
title: 文档标题
tags: [技术, Vue, 前端]
category: 开发文档
created: 2026-03-05
updated: 2026-03-05
status: 进行中
priority: 高
---
```
- 可视化属性编辑器（侧边栏卡片）
- 基于属性的筛选和搜索
- 导出时保留元数据

**技术实现**：
- YAML解析器（前端）
- 属性验证

### 2.3 标签分类系统 ⭐⭐⭐⭐
**用户需求**：多维度组织文档
**功能描述**：
- 文件右键菜单"添加标签"
- 侧边栏标签云显示
- 按标签筛选文件树
- 标签颜色自定义
- 批量标签管理

**技术实现**：
- 标签存储到单独索引文件
- 与Front Matter系统集成

### 2.4 内容片段（Snippet）库 ⭐⭐⭐
**用户需求**：复用常用代码块和文本模板
**功能描述**：
- `!snippet(名称)`快速插入
- 片段分类管理（SQL、Shell、配置等）
- 支持变量占位符
- 导入/导出片段库

## 🛠️ 第三梯队：专业工具集成

### 3.1 Git版本控制可视化 ⭐⭐⭐⭐⭐
**用户需求**：无需离开编辑器管理Git
**功能描述**：
- 集成Git操作界面
- 可视化提交历史时间线
- 文件状态标记（已修改、未跟踪等）
- 内嵌Diff对比（彩色高亮）
- 一键提交、推送、拉取
- 分支管理

**技术实现**：
- 使用`git2-rs`或调用系统Git
- 状态缓存和更新

### 3.2 写作目标与进度跟踪 ⭐⭐⭐⭐
**用户需求**：管理写作任务，保持动力
**功能描述**：
- 设置写作目标（字数/时间）
- 实时进度条显示
- 番茄钟专注计时器
- 每日/每周写作统计图表
- 成就系统（连续写作天数等）

### 3.3 参考文献管理 ⭐⭐⭐⭐
**用户需求**：学术写作参考文献管理
**功能描述**：
- 参考文献数据库
- `[@cite2023]`引用格式
- 支持APA、MLA、GB/T 7714等格式
- 自动生成参考文献列表
- 从DOI、ISBN自动获取文献信息

### 3.4 文档结构分析 ⭐⭐⭐
**用户需求**：优化文档结构质量
**功能描述**：
- 标题层级分析（建议H1→H2→H3顺序）
- 段落长度分布图
- 关键词密度分析
- 可读性评分（Flesch-Kincaid等）
- 结构优化建议

## 🌐 第四梯队：发布与分享

### 4.1 一键多平台发布 ⭐⭐⭐⭐
**用户需求**：内容分发到多个平台
**功能描述**：
- 支持平台：博客园、知乎、CSDN、掘金、WordPress
- 平台API配置管理
- 自动转换图片链接为平台格式
- 定时发布
- 发布状态跟踪

### 4.2 静态网站生成器 ⭐⭐⭐⭐
**用户需求**：将知识库转为个人网站
**功能描述**：
- 选择主题（多种预设）
- 自动生成导航菜单
- 支持搜索功能
- 一键部署到GitHub Pages/Vercel/Netlify
- 自定义域名配置

### 4.3 导出为电子书（EPUB） ⭐⭐⭐
**用户需求**：制作电子书分享知识
**功能描述**：
- 选择多篇文档打包
- 设置封面、作者、简介
- 自动生成目录
- 优化电子书排版
- 支持EPUB 3.0标准

## 🤖 第五梯队：智能处理

### 5.1 文档自动化处理流水线 ⭐⭐⭐⭐
**用户需求**：批量处理文档，标准化格式
**功能描述**：
- 自定义处理步骤：
  1. 清理多余空行
  2. 标准化标题格式（# 后加空格）
  3. 优化图片尺寸标记
  4. 检查并修复死链
  5. 拼写检查
- 对整个文件夹执行
- 保存处理配置

### 5.2 图片智能处理 ⭐⭐⭐⭐
**用户需求**：优化插入的图片
**功能描述**：
- 粘贴时自动压缩图片
- 添加水印（可选）
- 调整到合适尺寸
- 格式转换（PNG→WebP）
- OCR识别图片文字（集成Tesseract）

### 5.3 内容审查与合规检查 ⭐⭐⭐
**用户需求**：确保内容合规性
**功能描述**：
- 敏感词检测（可配置词库）
- 隐私信息脱敏（身份证、手机号等）
- 版权检查（引用标注）
- 生成合规性报告

## 🔧 技术架构考虑

### 前端（Vue 3 + TypeScript）
- 组件化设计，确保新功能可插拔
- 状态管理扩展（Pinia模块）
- 响应式设计适配新UI

### 后端（Rust + Tauri 2.0）
- 命令式API设计，保持现有模式
- 性能敏感操作使用Rust（编码转换、全文搜索等）
- 配置向后兼容

### 数据存储
- 新增功能使用独立配置文件
- 避免破坏现有数据结构
- 提供数据迁移工具（如需要）
