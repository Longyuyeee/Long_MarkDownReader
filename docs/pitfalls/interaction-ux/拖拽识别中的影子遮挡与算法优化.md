# 坑点：穿透式拖拽识别算法

> **技术关键词**：Drag and Drop, Hit Testing, document.elementsFromPoint, 指针事件 (Pointer-events), 穿透探测

## 1. 影子 DOM 遮挡问题
**现象**：拖拽影子（Ghost）显示正常，但鼠标悬停在文件夹上时，文件夹不显示高亮，松手也无反应。
**原因**：默认情况下，鼠标下方的第一个元素是“拖拽影子”本身。即使设置了 `pointer-events: none`，由于它紧贴鼠标尖，在某些浏览器刷新帧中仍会干扰探测。
**方案**：
- **物理偏移**：将影子坐标设置 `+10px` 的偏移，确保它不处在鼠标尖的正下方。
- **递归探测**：使用 `document.elementsFromPoint(x, y)` 获取该点下所有的元素数组，循环排除掉 `classList.contains('drag-ghost')` 的项。

## 2. 文件夹与视口识别冲突
**现象**：由于文件树很大，鼠标移动时总是优先命中“侧边栏视口（Viewport）”，导致识别为移动到根目录。
**原因**：判定优先级逻辑有误。
**方案**：
- **节点优先**：在元素数组中，一旦命中带有 `data-drop-path` 的树节点，立即锁定并 `break` 循环。
- **文件回溯**：如果命中文件，自动取其父目录。只有在扫描完所有元素都没发现节点时，才判定为“视口空白处”。
