> **技术关键词**：Vue 3 Reactivity, watch, recursion, Race Condition

### 现象
进入设置页面后 CPU 占用飙升，应用卡死，控制台频繁打印保存日志。

### 原因分析
在 `onMounted` 钩子中调用 `loadConfig()` 会异步更新 `config` 响应式对象。由于设置页面对 `config` 开启了 `{ deep: true }` 的 `watch` 监听，数据的更新立刻触发了 `updateConfig()` -> `save_config`。
如果 `updateConfig` 中又包含了对状态的校准或回写逻辑，就会形成 `Load -> Change -> Watch -> Update -> Save -> Reload` 的无限递归。

### 解决方案
引入 **初始化锁 (Initialization Lock)**。
1. 定义 `isInitializing` 状态位。
2. 在 `loadConfig` 开始前置为 `true`，完成后通过 `nextTick` 恢复为 `false`。
3. 在 `watch` 回调中检查锁状态，若处于初始化阶段则直接 `return`。
