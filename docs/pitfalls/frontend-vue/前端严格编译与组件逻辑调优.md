# 坑点：前端严格编译与组件逻辑冲突

> **技术关键词**：TypeScript (TS), vue-tsc, Naive UI, n-tree, 严格模式, 生产构建 (Build)

## 1. 生产构建的 TS 严格检查
**现象**：`npm run dev` 运行正常，但 `npm run tauri build` 报错中止。
**原因**：生产打包会触发 `vue-tsc --noEmit`。此时任何“已声明但未使用”的变量（TS6133）或“参数个数不匹配”（TS2554）都会被视为错误。
**方案**：在发布前必须先运行一次 `npm run build`。移除所有未使用的函数参数，并严格校准函数调用。

## 2. Naive UI n-tree 的多选逻辑冲突
**现象**：开启 `multiple` 属性后，单击节点会自动进入多选切换模式（Toggle），无法实现 Windows 风格的点选。
**原因**：组件原生的 `multiple` 逻辑与文件管理器的操作习惯不符。
**方案**：
- 移除 `v-model:selected-keys` 的双向绑定。
- 在 `nodeProps` 的 `onClick` 中通过 `e.ctrlKey` 手动维护 `selectedKeys`。
- 确保普通点击时重置数组，只有 Ctrl 点击时才执行 `push/splice`。
